# 事件流界面设计规范

> 适用于 `astrbot-plugin-enhanced-memory` 项目 WebUI 中的 `事件流（Event Flow）` 页面。
> 本文档描述交互逻辑、布局结构与实现要点，供 Claude Code 在修改 `web/static/` 原项目代码时参考。

## 2. 主时间轴（Main Axis）

- 位置：Canvas 最左侧，固定 `x = 28px`（相对 canvas 左边缘）
- 样式：`stroke: rgba(255,255,255,0.15)`, `stroke-width: 1.5`
- **无箭头**，仅一条竖线
- 每个时间行（row）在主轴旁有一个小刻度 tick（`±3px`）
- 刻度旁显示日期标签（`MM/DD` 格式，`JetBrains Mono`, `9px`, `rgba(255,255,255,0.2)`）

---

## 3. 节点行（Row）排布逻辑 ⚠️ 重要

**核心原则：Y 轴按事件发生顺序等距排列，而非时间等比缩放。**

### 实现步骤

1. 收集所有可见线程的全部事件，按 `timestamp` 升序排列
2. 依次分配 `rowIndex`（从 0 开始整数递增）
3. 时间差 ≤ 2小时（`MERGE_MS = 7200000ms`）的相邻事件视为**同行**，共享同一 `rowIndex`
4. 每行的 Y 坐标：`rowY = TOP_PAD + rowIndex × ROW_H`
   - `TOP_PAD = 40px`
   - `ROW_H = 68px`（固定，可配置）

### 代码参考

```js
const MERGE_MS = 2 * 3600 * 1000;

function buildRowMap(threads) {
  const allEvts = threads.flatMap(t =>
    t.events.map(ev => ({ ...ev, tsMs: ev.ts.getTime() }))
  ).sort((a, b) => a.tsMs - b.tsMs);

  const rows = [];
  const eventRowMap = {};
  allEvts.forEach(ev => {
    const last = rows[rows.length - 1];
    if (last && ev.tsMs - last.tsMs <= MERGE_MS) {
      eventRowMap[ev.id] = last.rowIdx;
    } else {
      const idx = rows.length;
      rows.push({ rowIdx: idx, tsMs: ev.tsMs });
      eventRowMap[ev.id] = idx;
    }
  });
  return { eventRowMap, totalRows: rows.length, rows };
}
```

## 4. 事件线程（Event Threads）

### 水平位置

```js
const FIRST_TX   = 58;   // 第一条线程的 x 坐标
const THREAD_W   = 14;   // 线程列宽
const THREAD_GAP = 16;   // 线程间间距

threadX(ti) = FIRST_TX + ti * (THREAD_W + THREAD_GAP)
```

线程从左到右紧密排列，不占用太多横向空间，右侧留给气泡展开。

### 每条线程包含

| 元素 | 描述 |
|------|------|
| **垂直线** | 从第一个节点 row 到最后一个节点 row，`stroke-width: 1.5`, `opacity: 0.7` |
| **节点圆圈** | `r=5`，默认空心（fill=bg），hover 时填充线程色，`r` 增大至 6.5 |
| **Glow ring** | hover 激活时在节点外额外绘制一个半透明大圆（`r=10, opacity=0.08`），增强聚焦感 |
| **结束标记** | 最后一个节点下方 18px 处，一条短横线（`±8px`）|
| **虚线连接器** | 从结束标记左端延伸到主轴，`stroke-dasharray: 3 2`，默认 `opacity: 0.15`，激活时 `0.6` |
| **主轴刻度** | 主轴上对应结束标记 Y 位置的小横线，激活时 `opacity: 0.9` |

## 5. 交互逻辑

### 5.1 Hover 聚焦

**触发**：鼠标进入某条线程列的不可见 hover zone（宽 `THREAD_W + THREAD_GAP`，高覆盖节点范围）

**效果**（同时发生）：

1. **当前线程** → `opacity: 1`，节点填充，显示 glow ring
2. **其他线程** → `opacity: dimOpacity`（默认 `0.1`，可通过 Tweaks 调节）
3. **气泡卡片** → 在当前线程所有节点右侧展开（见第 6 节）
4. **虚线连接器** → 当前线程的连接器 opacity 提升至 `0.6`

```js
// Opacity 计算优先级
function getOpacity(thread) {
  const manuallydimmed = dimmedIds.has(thread.id);
  if (hoveredThread !== null) {
    return hoveredThread === thread.id ? 1 : dimOpacity;
  }
  if (manuallydimmed) return dimOpacity;
  return 1;
}
```

### 5.2 Tag 多选筛选

- Tag chips 支持**多选**（Toggle 行为，已选再点取消）
- 选中集合 `activeTags: Set<string>`
- 匹配逻辑：线程的任意事件包含任意选中 tag → 该线程可见

```js
const matchedIds = new Set(
  ALL_THREADS
    .filter(t => t.events.some(ev => ev.tags.some(tag => activeTags.has(tag))))
    .map(t => t.id)
);
```

- `activeTags` 为空时 → 所有线程可见
- 切换 tag 时重置 `dimmedIds`

### 5.3 Thread Toggle（事件轴开关）

- Thread chip **只有**在 `matchedIds` 中的线程才会亮起（可点击）
- 未命中的线程 chip 置灰，`opacity: 0.38`，`cursor: default`（不可交互）
- 点击亮起的 chip → 将该线程 id 加入/移出 `dimmedIds`
- `dimmedIds` 中的线程：在时间轴上仍然**存在**，但 `opacity = dimOpacity`

**两层控制分离**：
```
activeTags  →  决定哪些线程出现（matchedIds）
dimmedIds   →  决定各线程的可见程度（opacity）
hover       →  临时覆盖所有 opacity
```

## 6. 气泡卡片（Bubble Card）

**出现时机**：仅在 hover 激活的线程上，为该线程的**每个节点**各渲染一张卡片

**位置**：
```
left = threadX(ti) + NODE_R + BUBBLE_GAP   // BUBBLE_GAP = 14px
top  = rowY(eventRowMap[ev.id]) - 24px
```

**样式**：
```css
background: rgba(30, 41, 59, 0.95);   /* --panel */
border: 1px solid rgba(255,255,255,0.08);
border-left: 2px solid <threadColor>;  /* 彩色左边框标识线程 */
border-radius: 8px;
backdrop-filter: blur(12px);
box-shadow: 0 4px 16px rgba(0,0,0,0.35);
animation: bubbleIn 0.14s ease;
```

**卡片内容**：
| 字段 | 样式 |
|------|------|
| 事件标题 | `12px`, `font-weight: 600`, `color: --text` |
| 时间戳 | `10px`, `JetBrains Mono`, `color: --text-dim` |
| Tags | 小圆角 badge，`background: threadColor + 16`，`color: threadColor` |
| Salience | 右侧 badge，≥85% 用 `--warn`，≥70% 用 `--accent`，其他 `--text-dim` |

**连接线**：卡片左边有一条 1px 细线连向节点（`width: 9px, background: threadColor, opacity: 0.5`）

## 7. 折叠机制（CollapsibleRow）

Tag legend 和 Thread toggle bar 均使用同一折叠组件：

- 默认高度 = 单行高度（`36px`）
- 使用 `ResizeObserver` 测量内容自然高度
- 当 `innerHeight > 36 + 6px` 时，右侧出现「+N 更多 ▾」按钮
- 点击展开 → 高度动画过渡至 `innerHeight`（`transition: height 0.22s ease`）
- 再点收起

## 9. 原项目代码对接要点

### 需要修改的文件

| 文件 | 需要做的事 |
|------|-----------|
| `web/static/pages/events.html` | 重构 HTML 结构，增加 tag legend 和 thread toggle bar 容器 |
| `web/static/css/timeline.css` | 替换为新的布局和组件样式 |
| `web/static/js/timeline.js` | 重构 `renderTimeline()` 实现新的行排布逻辑和交互 |

### `renderTimeline()` 重构要点

1. **收集所有事件 → 排序 → buildRowMap** → 得到 `eventRowMap`
2. **绘制主轴**：SVG `<line>` 无箭头
3. **按线程绘制**：每条线程一个 `<g>` 元素，`opacity` 通过 JS 控制
4. **Hover 事件**：绑定在 thread 的不可见 div zone 上，触发时：
   - 设置 `activeThreadId`
   - 更新所有线程 `<g>` 的 opacity
   - 渲染气泡卡片（绝对定位 div，append 到 scroll container）
5. **Tag 筛选**：维护 `activeTags Set`，变化时重新调用 `renderTimeline()`

### API 数据字段映射

```js
// 原始 API 返回的 event 对象 → 新渲染所需字段
ev.id           → ev.id
ev.content      → ev.title（显示标题）
ev.start        → ev.ts（ISO string → new Date()）
ev.tags         → ev.tags（数组）
ev.salience     → ev.sal（0~1 浮点数）
ev.group        → 用于 group badge
ev.participants → 用于 participants badge
ev.inherit_from → 用于父子线程连线（bridge lines）
```

### 事件线程分组建议

原项目的事件是平铺的，没有显式的「线程」概念。建议的线程分组策略：

1. **按 `group` 字段分组**：同一群组的事件归为一条线程
2. **或按 `inherit_from` 链式关系**：将有父子关系的事件串成一条线程
3. **或按时间窗口聚类**：将时间相近、tag 相似的事件归为一条线程
